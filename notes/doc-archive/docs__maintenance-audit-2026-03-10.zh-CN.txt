## 2) 发现的脆弱点与修复

### A) UI 与游戏创建链路

- Builder path 已以 SSR+HTMX 完成项目创建与跳转。
- Publish 使用 `/api/builder/projects/:projectId/publish` 并返回可见校验详情。
- Play 按钮由 `publishedReleaseVersion` 绑定，避免直接运行 draft。
- `/game` 页面由服务端注入 bootstrapped session，再进入 runtime。

### B) 静态资源安全

- 解码后做路径遍历检查。
- 拒绝 `.`/`..` 片段。
- 读文件前先检查挂载目录前缀。
- Bun API 读取并进行存在性短路判断。

### C) 运行契约隔离

- 会话生命周期集中于 `game-loop` 与其 helper。
- 运行态读取仅来自发布快照。
- 所有失败通过信封化，保证 UI 具备确定性状态映射。

### D) 文档完整性

- 每个归档条目含 sourcePath、archivePath、sha256、sizeBytes、archivedAt。
- `docs:check` 校验 required keys，避免依赖真实 `.md` 文件。
