# Design System — Card Variants and Usage

## Container Width

Main content container: `max-w-[1600px]` for builder and general layout. Consider `max-w-7xl` (1280px) for tighter reading on large screens; 1600px retained for builder workspace.

## Card Variants (cardClasses)

| Variant | Classes | When to Use |
|---------|---------|-------------|
| default | card bg-base-100 shadow-sm | Standard cards without border. Use for simple content blocks. |
| bordered | card card-border bg-base-100 shadow-sm | Cards with visible border. Use when separation from background matters. |
| borderedGlass | card card-border bg-base-100/90 shadow-sm backdrop-blur-sm | Glassmorphism effect with blur. Use for overlays, headers, floating panels. |
| borderedGlassNoBlur | card card-border bg-base-100/90 shadow-sm | Semi-transparent without blur. Use when blur is undesirable but transparency is needed. |
| borderedNoShadow | card card-border bg-base-100 | Border only, no shadow. Use for flat, minimal cards. |

## Examples

- **default**: Status cards, simple lists, minimal chrome
- **bordered**: Form sections, content blocks, article cards
- **borderedGlass**: Game header, navbar, modal overlays
- **borderedGlassNoBlur**: Dashboard metric cards where blur would obscure content
- **borderedNoShadow**: Inline panels, nested cards

## Usage

Import from `src/views/shared/ui-components.ts`:

```ts
import { cardClasses } from "./shared/ui-components.ts";

// Apply variant
<div class="${cardClasses.borderedGlass}">...</div>
```

Or use `renderCard()` with `variant` option for structured cards with title, body, actions.
